# Security Policy

## Reporting a vulnerability

Do not open a public GitHub issue for suspected vulnerabilities.

Use GitHub private vulnerability reporting if it is enabled for this repository.
If private vulnerability reporting is not enabled, report the issue privately to
`aleksei@mancala.com`.

Please include:

- a short description of the issue
- affected files, commands, or workflows
- reproduction steps or proof of concept
- impact assessment
- any environment or configuration details needed to reproduce it

## Response expectations

This repository is still early, so response timing is best-effort rather than a
guaranteed SLA. Private reports are preferred so maintainers can investigate and
coordinate a fix before public disclosure.
